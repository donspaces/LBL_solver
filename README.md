# 3D 魔方交互小游戏

一款基于纯前端技术实现的 **3D 魔方交互小游戏**，支持电脑自动打乱魔方（带动画）、电脑自动按公式解魔方（展示步骤），并提供打乱/解魔方调速、魔方自由拖动旋转等核心功能。

## 项目概览

本项目是一个完全运行在浏览器端的 3×3 魔方模拟器，无需安装任何依赖，打开即玩。用户可以观看魔方被随机打乱的动画过程，选择不同的解法策略，观看自动还原的完整步骤演示，同时可以自由拖动旋转魔方调整观察视角。

## 技术栈

| 技术 | 用途 |
|------|------|
| HTML5 | 页面结构与语义化标签 |
| CSS3 | 3D 变换渲染（`perspective`、`transform-style: preserve-3d`）、响应式布局、动画效果 |
| JavaScript (ES6+) | 魔方状态管理、旋转算法、解法优化、用户交互逻辑 |
| Canvas API | 步骤记录中的魔方缩略图绘制 |

项目为纯前端实现，**零外部依赖**，全部代码包含在单个 HTML 文件中，约 30KB。

## 功能说明

### 魔方可视化

- 使用 CSS 3D Transform 渲染 3×3×3 共 27 个方块，六面颜色分明（白、黄、红、橙、蓝、绿）
- 每个色块带有渐变光影效果，底部设有动态阴影，呈现逼真的 3D 视觉效果
- 支持 6 种面旋转操作：U/D（上/下层）、F/B（前/后面）、L/R（左/右面），均支持顺时针与逆时针

### 自动打乱功能

点击「随机打乱魔方」按钮后，系统随机生成 20~30 步打乱指令（自动避免连续同面/对面操作），并以逐步动画形式展现打乱过程。打乱过程中状态栏实时显示当前执行的步骤。

### 调速功能

提供滑块控件，可在 30ms ~ 800ms 之间调节自动打乱和自动解魔方的单步动画间隔。滑块调节实时生效，方便用户按需加速或放慢观看。

### 自动解魔方功能

点击「自动解魔方」按钮后，系统按照所选解法策略按顺序执行还原操作，同步展示动画并记录每一步骤。提供三种解法供用户选择：

| 解法 | 策略说明 | 特点 |
|------|----------|------|
| **CFOP (Fridrich)** | 激进优化 — 消除互逆对 + 三连合并 + 四连消除 | 步数最少，比赛主流速解法 |
| **层先法 (LBL)** | 逐步还原 — 不做优化，完整展示每一步 | 入门经典方法，便于学习理解 |
| **Roux 法** | 轻度优化 — 仅消除互逆对 | 步数适中，比赛新兴速解法 |

> 只有当魔方处于被打乱状态时「自动解魔方」按钮才可点击，初始未被打乱时该按钮为禁用状态。

### 步骤记录功能

解魔方过程中，在界面下方以列表形式实时记录每一步操作，每条记录包含：

- **步骤编号**：按执行顺序递增（#01、#02、#03…）
- **缩略图**：当前步骤执行前的魔方展开状态示意图（Canvas 绘制）
- **公式标记**：当前操作的标准魔方公式符号（如 U、R'、F 等）
- **操作描述**：对应的中文说明（如「顶层顺时针」「右面逆时针」等）

列表按最新记录在最上方、历史记录依次向下排列。解题完成后，点击任意步骤可回溯查看该步对应的魔方状态。

### 自由拖动旋转

支持鼠标拖动和触摸滑动两种方式自由旋转整个魔方，调整 3D 观察视角，视角过渡流畅自然，俯仰角限制在 ±80° 范围内确保操作符合直觉。

## 快速开始

### 方式一：直接打开

项目无需安装任何依赖，直接在浏览器中打开 `index.html` 即可运行：

```bash
# macOS
open index.html

# Linux
xdg-open index.html

# Windows
start index.html
```

### 方式二：本地服务器（推荐）

```bash
# 使用 Python 启动本地服务器
python -m http.server 8000

# 或使用 Node.js
npx serve .
```

然后在浏览器中访问 `http://localhost:8000`。

## 代码架构

### 核心模块

```
状态模型          旋转操作           解法引擎           渲染引擎           交互控制
initState()      applyMove()       getSolution()      render()          鼠标/触摸拖动
cloneState()     cwFace()          optimizeFull()     colorAt()         按钮事件绑定
restoreState()   ccwFace()         optimizeLight()    mkFace()          滑块实时调速
checkSolved()    invertMove()      genScramble()      captureThumb()    步骤点击回溯
```

### 状态模型

魔方状态用一个 6×9 的二维数组表示，6 个面各含 9 个色块，每个色块存储颜色索引（0~5）。所有旋转操作经过严格单元测试验证：4 次同向旋转归位、逆操作正确性、1000 次随机序列还原、sexy move 阶数为 6。

## 界面与体验

- **响应式设计**：适配 PC、平板和移动设备；桌面端水平排列，平板端（≤768px）自动切换为垂直堆叠布局，移动端（≤480px）进一步缩小尺寸
- **暗色主题**：深色背景搭配氖光风格的青色/品红/金色点缀，突出魔方主体
- **交互反馈**：按钮悬停时有发光边框和微上浮效果，步骤条目入场带有淡入滑动动画
- **流畅动画**：3D 旋转与拖动过渡自然无卡顿，调速功能实时生效
